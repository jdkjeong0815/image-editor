# 이미지 관리 및 업데이트 제안 (Image Management Proposals)

브라우저 기반의 이미지 에디터에서 `image/` 폴더 내의 새로운 이미지를 동적으로 반영하기 위한 4가지 제안 사항입니다.

## 1. Directory Picker (폴더 통째로 선택) - **추천**
*   **방식**: 사용자가 [폴더 선택] 버튼을 누르고 `image` 폴더를 지정합니다.
*   **특징**: 서버 개발 없이 브라우저 단에서 폴더 내 수백 장의 파일을 즉시 읽어 슬라이드쇼 명단을 만듭니다.
*   **장점**: 보안 위반 없이 로컬 파일을 가장 빠르고 완전하게 동적 반영할 수 있습니다.

## 2. 드래그 앤 드롭 (Batch Upload)
*   **방식**: 탐색기에서 다수의 이미지 파일을 선택하여 편집기에 끌어다 놓습니다.
*   **특징**: 드롭된 파일들을 브라우저 메모리상에서 순환 리스트(Pool)로 만듭니다.
*   **장점**: 사용법이 직관적이며 별도의 설정이 전혀 필요 없습니다.

## 3. 서버 사이드 자동화 (Full Automation)
*   **방식**: Node.js나 Python 백엔드 서버를 작게 구축합니다.
*   **특징**: 서버가 폴더를 실시간 감시(Watch)하며 파일 추가 시 브라우저에 즉시 알림을 보냅니다.
*   **장점**: 사용자가 개입할 필요 없이 폴더에 파일을 넣기만 하면 자동으로 반영됩니다.

## 4. 로컬 업데이트 스크립트 (CLI Tool)
*   **방식**: `update_list.py`와 같은 실행 파일을 바탕화면에 둡니다.
*   **특징**: 파일 추가 후 스크립트 실행 시 `image_list.json` 명단을 즉시 갱신합니다.
*   **장점**: 현재 구조를 그대로 유지하면서 가장 적은 코드 수정으로 운영 가능합니다.
