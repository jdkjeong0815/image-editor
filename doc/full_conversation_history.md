# Antigravity Studio - 전체 대화 및 개발 기록 (Full Conversation Log)

본 문서는 프로젝트 시작부터 현재까지의 모든 주요 요청 사항, 기술적 해결 방법, 그리고 구현된 기능들의 히스토리를 기록한 파일입니다.

---

## [Phase 1] 초기 시스템 구축 및 이미지 처리 기초
- **요청**: 이미지에 그레이스케일, 라인 아트 등의 효과를 슬라이더로 조절하는 프로그램 개발.
- **해결**: HTML5 Canvas와 JavaScript를 이용한 비파괴 편집 시스템 구축. `CamanJS` 라이브러리 없이 순수 Canvas API로 필터(밝기, 대비, 선명도 등) 구현.

## [Phase 2] 시네마틱 애니메이션 및 경로 설정 ( Cinematic Path)
- **요청**: 이미지 내 특정 지점들을 순환하며 보여주는 시네마틱 효과 추가.
- **해결**:
    - 사용자가 이미지 위를 클릭하여 포인트를 지정하는 기능 구현.
    - 지정된 포인트(1, 2, 3, 4)를 부드럽게 이동하는 `requestAnimationFrame` 기반 엔진 구축.
    - 지점마다 3초 정지 및 마지막 '전체 보기' 단계 추가.
    - **줌 보정**: 지점 간 이동 시 카메라가 멀어졌다(줌아웃) 가까워지는(줌인) 입체적 연출 적용.

## [Phase 3] 시네마틱 속도 및 디테일 최적화
- **요청**: 애니메이션 속도를 극도로 늦추고(-15배), 줌 배율(2.5배) 조정 시 픽셀 깨짐 방지.
- **해결**:
    - **속도 보정**: 슬라이더 범위를 ±5초로 재설정하고, 내부 연산을 통해 베이스 속도를 대폭 하향 조정.
    - **고해상도 대응**: webp 형식으로 이미지를 최적화하여 줌인 시에도 디테일 유지.
    - **경계 보호**: 확대 시 이미지 바깥의 검은 공백이 보이지 않도록 `Clamping` 로직 적용.

## [Phase 4] 듀얼 모니터 및 전시 시스템 (Dual Monitor Display)
- **요청**: 메인 모니터에는 메뉴를, 확장 모니터에는 이미지만을 전체 화면으로 송출.
- **해결**:
    - `window.open`을 이용한 독립 송출창(`display.html`) 구현.
    - **Aspect-Correct Letterbox**: 모니터 해상도에 맞춰 이미지를 최대로 키우되, 원본 비율을 깨뜨리지 않는 검은 여백(레터박스) 로직 적용.
    - **원클릭 전체화면**: 클릭 시 브라우저 UI가 사라지는 Fullscreen API 연결.
    - **실시간 동기화**: 메인 창의 필터와 애니메이션 상태를 확장 창에 즉각 복제.

## [Phase 5] 자동화 및 슬라이드쇼 (Automated Slideshow)
- **요청**: `image/` 폴더 내의 모든 이미지를 무한 반복하며 자동으로 슬라이드쇼 재생.
- **해결**:
    - 파이썬 스크립트를 통해 폴더 내 이미지를 스캔하여 `image_list.json` 명단 작성.
    - 시네마틱 루프가 1회 완료될 때마다 다음 이미지로 자동 전환되는 동적 로드 시스템 구축.

## [주요 기술 스택 및 파일 구조]
- **HTML/CSS**: `index.html`, `display.html` (모던 다크 테마 및 유리 효과 디자인)
- **JavaScript**: `script.js` (애니메이션 엔진, 필터 로직, 윈도우 동기화)
- **Python**: `led_optimizer.py` (이미지 최적화 및 명단 관리)
- **Data**: `image_list.json` (슬라이드쇼 대상 이미지 리스트)
